## 1. PRESUPUESTOS (BUDGETS)

### 1.1 Modelo de Datos

**Tabla: `budgets`**
```sql
CREATE TABLE budgets (
  budget_id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(user_id),
  category_id INTEGER NOT NULL REFERENCES categories(category_id),
  amount DECIMAL(15,2) NOT NULL, -- Presupuesto mensual o anual
  period VARCHAR(10) NOT NULL CHECK (period IN ('monthly', 'yearly')), -- 'monthly' o 'yearly'
  start_date DATE, -- Fecha de inicio (opcional, si no se especifica, usar fecha actual)
  end_date DATE, -- Fecha de fin (opcional, si no se especifica, presupuesto indefinido)
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, category_id, period, start_date) -- Un presupuesto √∫nico por categor√≠a, per√≠odo y fecha de inicio
);
```

### 1.2 Endpoints Requeridos

#### GET `/api/users/{userId}/budgets/status`
Obtener el estado de todos los presupuestos (gastado vs presupuestado).

**Query Parameters (opcionales):**
- `startDate` (string, formato: `YYYY-MM-DD`): Fecha de inicio del per√≠odo a analizar
- `endDate` (string, formato: `YYYY-MM-DD`): Fecha de fin del per√≠odo a analizar

**L√≥gica:**
- Si `startDate` y `endDate` no se proporcionan, usar el mes actual
- Para cada presupuesto:
  - Si `period = 'monthly'`: calcular gastos del mes actual (o del mes especificado por startDate/endDate)
  - Si `period = 'yearly'`: calcular gastos del a√±o actual (o del a√±o especificado)
- Calcular `spentAmount` sumando todas las transacciones de tipo `expense` de esa categor√≠a en el per√≠odo
- Calcular `percentageUsed = (spentAmount / budgetAmount) * 100`
- `isExceeded = spentAmount > budgetAmount`

```json

    "budgetId": 1,
    "categoryId": 5,
    "categoryName": "Comida",
    "budgetAmount": 500.00,
    "spentAmount": 350.00,
    "remainingAmount": 150.00,
    "percentageUsed": 70.0,
    "isExceeded": false,
    "period": "monthly",
    "startDate": "2024-01-01",
    "endDate": "2024-01-31"
  }

```

## 3. NOTAS IMPORTANTES

### 3.1 Autenticaci√≥n
Todos los endpoints requieren autenticaci√≥n mediante Bearer Token en el header `Authorization`.

### 3.2 Validaci√≥n de Usuario
Todos los endpoints deben validar que:
- El `userId` en la URL corresponde al usuario autenticado
- Los recursos (categor√≠as, activos, etc.) pertenecen al usuario

### 3.3 Manejo de Errores
- **400 Bad Request**: Datos inv√°lidos (amount <= 0, period inv√°lido, etc.)
- **401 Unauthorized**: Token inv√°lido o expirado
- **403 Forbidden**: Usuario no tiene permiso para acceder al recurso
- **404 Not Found**: Recurso no encontrado (budgetId, templateId, etc.)
- **409 Conflict**: Conflicto (ej: presupuesto duplicado para la misma categor√≠a/per√≠odo)

### 3.4 Formato de Fechas
- Todas las fechas deben estar en formato ISO 8601: `YYYY-MM-DD` o `YYYY-MM-DDTHH:mm:ssZ`
- Para presupuestos mensuales, si no se especifica `startDate`, usar el primer d√≠a del mes actual
- Para presupuestos anuales, si no se especifica `startDate`, usar el primer d√≠a del a√±o actual

### 3.5 C√°lculo de Gastos en Presupuestos
- Solo contar transacciones de tipo `expense`
- Filtrar por `categoryId` exacto (no incluir subcategor√≠as a menos que se especifique)
- Para presupuestos mensuales: filtrar transacciones del mes especificado (o mes actual)
- Para presupuestos anuales: filtrar transacciones del a√±o especificado (o a√±o actual)
